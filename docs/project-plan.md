# EdgeLab Console Plan

## Product Positioning

Build a polished internal-tool style console that feels relevant to platform teams, lab teams, edge-device work, firmware validation, and operations-focused engineering groups.

## Core User Story

An engineer or technical lead needs to answer four questions quickly:

1. Which runs need attention right now?
2. What changed from baseline?
3. What artifacts and alerts came out of the run?
4. What should the next engineer know before taking over?

## MVP Scope

- Dashboard metrics
- Run list with filters
- Run detail page or panel
- Alerts, config drift, artifacts, and notes
- Live signal feed for the selected run

## What Makes It Portfolio-Worthy

- Full-stack architecture instead of a single-page frontend demo
- Domain modeled around real engineering workflow
- Strong operator UX with concise information hierarchy
- Public-safe but still technically credible
